สรุปการใช้งาน Docker Compose และ Cloud Services

Docker Compose

Docker Compose เป็นเครื่องมือที่ช่วยในการจัดการและรันหลายคอนเทนเนอร์ Docker พร้อมกันด้วยการใช้ไฟล์การตั้งค่า YAML คุณสามารถกำหนดบริการ เครือข่าย และโวลุ่มต่าง ๆ ในไฟล์เดียว และใช้คำสั่ง `docker-compose` เพื่อบริหารจัดการคอนเทนเนอร์ต่าง ๆ ได้ง่ายขึ้น

การติดตั้ง Docker Compose

การติดตั้ง Docker Compose สามารถทำได้ง่าย ๆ ด้วยคำสั่ง:

sudo curl -L "https://github.com/docker/compose/releases/download/1.29.2/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
docker-compose --version

การใช้งาน Docker Compose

ตัวอย่างไฟล์ docker-compose.yml

version: '3'
services:
  web:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./web:/usr/share/nginx/html
  db:
    image: postgres:alpine
    environment:
      POSTGRES_DB: example
      POSTGRES_USER: example
      POSTGRES_PASSWORD: example

คำสั่งที่ใช้ในการบริหารจัดการ Docker Compose:

  • docker-compose up - สร้างและเริ่มต้นคอนเทนเนอร์
  • docker-compose down - หยุดและลบคอนเทนเนอร์
  • docker-compose ps - แสดงสถานะของคอนเทนเนอร์
  • docker-compose logs - ดูล็อกของคอนเทนเนอร์

Cloud Services

Cloud Services เป็นการให้บริการระบบคอมพิวเตอร์ผ่านอินเทอร์เน็ต โดยสามารถเข้าถึงได้จากที่ใดก็ได้ ทำให้การบริหารจัดการทรัพยากรต่าง ๆ ง่ายขึ้น มีความยืดหยุ่นสูง และสามารถปรับขนาดได้ตามต้องการ

บริการ Cloud ที่นิยมใช้

  • Amazon Web Services (AWS): บริการคลาวด์จาก Amazon ที่มีบริการหลากหลาย เช่น EC2, S3, RDS
  • Google Cloud Platform (GCP): บริการคลาวด์จาก Google ที่เน้นการประมวลผลที่รวดเร็วและมีบริการหลายประเภท เช่น Compute Engine, Cloud Storage, BigQuery
  • Microsoft Azure: บริการคลาวด์จาก Microsoft ที่มีบริการหลากหลายและผสมผสานกับผลิตภัณฑ์อื่น ๆ ของ Microsoft ได้อย่างดี เช่น Virtual Machines, Azure Blob Storage, Azure SQL Database

การใช้งาน EC2 บน AWS

EC2 (Elastic Compute Cloud) เป็นบริการเครื่องเสมือน (Virtual Machine) ที่สามารถใช้ในการรันแอปพลิเคชันต่าง ๆ โดยคุณสามารถสร้างและบริหารจัดการเครื่อง EC2 ผ่าน AWS Management Console หรือใช้ AWS CLI (Command Line Interface)

aws ec2 run-instances --image-id ami-0abcdef1234567890 --count 1 --instance-type t2.micro --key-name MyKeyPair --security-groups my-sg
